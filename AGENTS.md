# aalmada.github.io - Agent Instructions

## Quick Reference
- **Stack**: Jekyll (Chirpy theme), Ruby/Bundler, GitHub Pages Actions
- **Entry points**: `_config.yml`, `_posts/`, `_plugins/posts-lastmod-hook.rb`
- **Common commands**:
  - `bundle install`
  - `bundle exec jekyll s`
  - `JEKYLL_ENV=production bundle exec jekyll b`
- **Docs**:
  - `README.md`
  - `https://github.com/cotes2020/jekyll-theme-chirpy#documentation`

## Repository Map
- `_posts/` -> All long-form content; frontmatter conventions matter for SEO/cards/redirects.
- `_plugins/posts-lastmod-hook.rb` -> Computes `last_modified_at` from git history at build time.
- `.github/workflows/pages-deploy.yml` -> Canonical Pages build/deploy behavior.

## Major Patterns
- Post URLs are generated by `permalink: /posts/:title/` in `_config.yml`; filename changes can break inbound links.
- `redirect_from` requires `jekyll-redirect-from` enabled in both `Gemfile` and `_config.yml` plugins.
- Last-modified metadata is not handwritten: plugin sets `last_modified_at` from `git log` when commit count > 1.
- CI uses `fetch-depth: 0` specifically so plugin git history lookups work.

## Code Rules (MUST follow)
```
✅ Keep `fetch-depth: 0` in Pages workflow         ❌ Use shallow checkout in deploy job
✅ Keep post permalink style stable (`/posts/:title/`) ❌ Change permalink format without redirect plan
✅ Keep `jekyll-redirect-from` enabled if using `redirect_from` ❌ Add `redirect_from` but disable the plugin
✅ Let plugin set `last_modified_at` from git      ❌ Hardcode `last_modified_at` in every post by default
✅ Ask user before editing AGENTS/SKILL guidance   ❌ Silently rewrite guidance files
```

## Common Mistakes
- Updating only `README.md` and expecting a deploy. Current workflow ignores README-only pushes.
- Refactoring deployment checkout depth and breaking `last_modified_at` generation.
- Changing permalink behavior and forgetting redirects for existing post URLs.

## Quick Troubleshooting
- Symptom: `last_modified_at` missing for older posts in CI.
  - Fix: verify `.github/workflows/pages-deploy.yml` still uses full history checkout.
- Symptom: post URL changed unexpectedly after rename.
  - Fix: preserve old path with `redirect_from` in the post frontmatter.
- Symptom: social preview shows site default image for a valid-looking post URL.
  - Fix: verify the shared URL resolves to the canonical post path (case-sensitive), then add a targeted `redirect_from` alias if needed.
- Symptom: local build differs from Pages output.
  - Fix: build with `JEKYLL_ENV=production bundle exec jekyll b`.

## Maintenance Note
When introducing a new non-obvious convention, update the relevant `AGENTS.md` in the same change. If a related `SKILL.md` or another guidance file is stale, propose an update, but ask the user for permission before editing guidance files.
